# Automation Requirement

## Rule

Final gameplay must rely on ordinary Stellaris mod scripting and local Strategic Nexus tooling.

Manual debug transport is not a release workflow, and the project must not build hidden automation around debug interfaces.

## Safe Automation Model

Production gameplay now uses the offline companion model described in `OFFLINE_CAMPAIGN_ANALYSIS_ARCHITECTURE.md`.
The model below remains valid as a local validation and development-harness boundary, not as a requirement for live LLM delivery into a running game.

The supported automation boundary is:

```text
host/user starts Strategic Nexus daemon
    -> daemon waits for explicit local work
    -> daemon writes bounded JSON recommendation
    -> bridge core validates payload and writes allowlisted artifact
    -> scripted mod receiver or development harness applies ordinary mod-side state
```

The daemon may run in the background, but the game must never block on it.

## Required Properties

Automation must be:

* local-first
* explicit
* bounded by schema
* validated before use
* replay-protected
* fail-closed
* compatible with normal scripted fallback behavior

## Forbidden Paths

Do not build:

* lower-level modification of the running game process
* tooling that attaches to, traces, patches, or inspects the running game
* proprietary executable-internal dispatch paths
* hidden debug-interface automation
* arbitrary command execution from daemon output
* arbitrary script text generated by an LLM
* custom host-to-client networking before normal multiplayer synchronization is disproven

Do not pass:

* raw LLM text
* arbitrary script text
* debug-interface command strings from daemon
* fleet targets
* build orders
* save fragments
* direct tactical commands

## Current Bridge-Core Transport Artifact

The bridge core may generate an allowlisted effect batch from validated `strategy_dimensions`.

Example:

```text
effect set_global_flag = strategic_nexus_bridge_available
effect set_global_flag = strategic_nexus_military_posture_defensive
effect set_global_flag = strategic_nexus_research_bias_economy
effect set_country_flag = strategic_nexus_bridge_available
effect set_country_flag = strategic_nexus_military_posture_defensive
effect set_country_flag = strategic_nexus_research_bias_economy
event strategic_nexus.1400
```

This is a local validation artifact, not raw daemon command execution.

The daemon still outputs only structured JSON. The bridge core validates it. The effect batch emitter selects only predefined flags and events from an allowlist.

Invalid payloads must clear stale output so an old batch is not accidentally reused.

## Multiplayer Constraint

Clients must not run LLM inference or generate local strategic decisions.

For multiplayer experiments, all gameplay-affecting mod files must be identical before game launch so checksum and state remain compatible. Host-generated decisions must be expressed through normal synchronized scripted game state.

## Safety Gate

When in doubt, prefer:

```text
documented mod script
local JSON
offline validation
manual development harness
```

Do not add lower-level game integration work to this project.
